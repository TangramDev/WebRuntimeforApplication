Common Web Runtime

Common Web Runtime(CWR) is the dynamic link library version of Google Chromium Project

       The CWR contains all the code for the chromium project. Its goal is to make the windows desktop application implement the built-in modern browser model according to the consistent scheme, and make the host desktop software become the modern web browser naturally. Different from the traditional browser, the application integrated with CWR has its own application object model. Because all functions of browser are provided based on DLL mode, CWR does not know what the host application is and what object model the host application contains. CWR solves the connection problem between host model and browser standard model, the real browser model can only be determined after a specific host application is defined.

       The DOM model of the host application will be a superset of the DOM model of the standard browser. In other words, the host application system will provide parts of the application model that cannot be provided in the standard browser. The actual DOM model of the host application may include members that are not part of the standard browser DOM model, which means that the host application supports both standard and application based web pages (there is no application based web page in standard browser mode). Because the host application has all the functions of chromium project built in, application developers can develop new application content for host application based on Web technology, and make host application become software ecosystem based on Web technology.

       In terms of DOM, CWR adds the support of. Net framework, which allows DOM elements for. Net UI to be written with DOM rules on Web pages. That is to say, WinForm, UserControl, WPF and other objects can be embodied in the actual development as part of page elements, which is very important for the formation of application ecology.
