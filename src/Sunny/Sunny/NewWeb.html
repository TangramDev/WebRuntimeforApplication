Dock: The attribute of being familiar and unfamiliar

Every .NET UI (WinForm, Usercontrol, WPF) object has a Dock property, which has always existed since the birth of the .NET Framework. Over the years, we have become accustomed to the mature use of the Dock attribute, and our thinking has fallen into inertia, forming a blind spot. Generally speaking, the Dock property divides UI objects into two categories, as shown in the table on the right.

About Dock Attribute Controls are classified by Dock Attribute

Because the Dock attribute is the second type of control, it must depend on a specific control. This dependency determines that this type of control generally cannot be the main bearing control of a certain type of content.

The first type of control includes the following two situations:

  • The control itself is a child object of another .NET UI control
  • Is Another Child Window of a non-.NET UI object (DockStyle.None)
